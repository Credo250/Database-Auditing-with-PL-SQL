# 🏦 Oracle Database Auditing System with PL/SQL

![Oracle](https://img.shields.io/badge/Database-Oracle-red)
![PL/SQL](https://img.shields.io/badge/Language-PL%2FSQL-blue)
![Triggers](https://img.shields.io/badge/Feature-Triggers-green)
![License](https://img.shields.io/badge/License-MIT-lightgrey)

A complete **Database Auditing System** developed in **Oracle PL/SQL** for monitoring data modifications in a banking-style database.

This project demonstrates how database triggers and stored procedures can be used to automatically record every **INSERT**, **UPDATE**, and **DELETE** operation while maintaining an audit trail and logging execution errors.

---

#  Project Overview

This project was developed as a **Database Programming** group assignment at the **University of Lay Adventist of Kigali (UNILAK)**.

The objective is to implement a reusable auditing framework capable of recording changes made to important database tables.

The system includes:

- Generic audit table
- Audit triggers
- Error logging
- Stored procedures
- Exception handling
- Testing scripts

---

# Objectives

The system automatically:

- Logs every INSERT operation
- Logs every UPDATE operation
- Logs every DELETE operation
- Records the database user performing the operation
- Records timestamps
- Stores old and new row values
- Handles runtime errors safely
- Maintains an independent error log

# Audit Tables

## AUDIT_LOG

Stores every database modification.

| Column | Description |
|---------|-------------|
| audit_id | Primary Key |
| operation_type | INSERT / UPDATE / DELETE |
| object_name | Table affected |
| operation_user | Database user |
| operation_timestamp | Time of operation |
| old_data | Data before modification |
| new_data | Data after modification |

---

## ERROR_LOG

Stores errors generated by stored procedures.

| Column | Description |
|---------|-------------|
| error_id | Primary Key |
| procedure_name | Procedure generating the error |
| error_message | Oracle error message |
| error_code | Oracle error number |
| error_timestamp | Time of error |

#  Audit Triggers

The project contains audit triggers for the following tables:

- ALLOWANCES
- ATTENDANCE
- COUNTRIES
- DEPARTMENTS
- EMPLOYEES
- ROLES

Each trigger automatically records:

- Operation Type
- Table Name
- User
- Timestamp
- Old Values
- New Values

---

#  Stored Procedures

Two exception-safe procedures are implemented.

## SP_UPDATE_ALLOWANCE

Updates an allowance record.

Features:

- Checks whether the record exists.
- Updates the allowance.
- Commits successful transactions.
- Logs errors automatically.

---

## SP_DELETE_ALLOWANCE

Deletes an allowance record.

Features:

- Checks whether the record exists.
- Deletes the allowance.
- Logs errors.
- Rolls back failed transactions.

# How to Run

1. Open Oracle SQL Developer or SQL*Plus.

2. Connect to your Oracle Database.

3. Execute:

```sql
@audit_system.sql
```

or run the script directly inside SQL Developer.

The script will automatically:

- Create all tables
- Create triggers
- Create stored procedures
- Insert sample data
- Run test cases
- Generate audit records
- Generate error logs

---

#  Expected Output

After execution:

- AUDIT_LOG contains every INSERT, UPDATE and DELETE operation.

- ERROR_LOG contains any handled exceptions generated by the stored procedures.
